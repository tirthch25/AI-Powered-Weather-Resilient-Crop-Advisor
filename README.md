## 🧭 Overview

The **AI Powered Weather Resilient Crop Advisor v4.0** is a globally scalable, fully agentic agricultural intelligence platform covering all **195 UN-recognised countries**. It is the first version with **zero static data** — every piece of information shown to the farmer (temperature, forecast, soil type, market prices, crop recommendations) is sourced from a live API or generated by a real-time LLM call.

A user selects their country, state, and district. The system runs a **5-step streaming pipeline**:
1. **Location Agent** — Resolves exact latitude/longitude for any global location
2. **Live Weather** — Open-Meteo current conditions (real-time, free API)
3. **6-Month Forecast** — Open-Meteo Archive API (real 2-year historical climatology)
4. **AI Soil & Market Enrichment** — Gemini fast enrichment (parallel) + search-grounded background upgrade
5. **Crop Agent** — LLM-powered ranking using Gemini + Google Search Grounding

## 🆕 What's New in v4.0

### Zero Static Data — Everything is Dynamic

| Removed (v3.x) | Replaced with (v4.0) |
|----------------|----------------------|
| `_ZONE_CLIMATE` — 120-cell static climate tables | **Open-Meteo Archive API** — real 2-year historical monthly averages per lat/lon |
| `_COUNTRY_TO_ZONE` — 30 country→zone mappings | Gemini LLM + lat-based fallback |
| `_ZONE_SOIL_DEFAULTS` — hardcoded soil per zone | Gemini `_llm_enrich_fast()` + search-grounded background upgrade |
| `_ZONE_MARKET_TEMPLATES` — templated market prices | Gemini Search Grounding — real current prices or nothing |
| `_COUNTRY_CURRENCY` — 30-country currency map | LLM returns correct local currency automatically |
| `_COUNTRY_CROP_HINTS` — 60+ country static hints | Gemini/Ollama prompt with dynamic regional context (no hints needed) |
| `_fallback_crops()` — 600+ line static crop table | `_llm_simple_fallback()` — minimal Gemini call, empty on failure |
| `_HINDI_CROP_NAMES`, `_SH_COUNTRIES` — static validators | Geographic region prompting makes these unnecessary |
| Zone-based weather estimate when API fails | `_llm_estimate_current_weather()` — LLM estimate from coordinates |
| 7-month forecast (bug) | **Exactly 6 months** |
| Hardcoded `25.0°C` in streaming endpoint | **Live weather temperature** used |

### New: Fast + Rich Parallel Soil Enrichment

The streaming pipeline now uses a two-stage soil enrichment strategy:
1. **`_llm_enrich_fast()`** — fires in parallel with the weather fetch, completes in ~5-8s, gives immediate soil/market data
2. **`/api/enrich-soil`** — background fetch after dashboard renders, runs full search-grounded Gemini for richer data, updates soil card smoothly

### Fixed: Null Weather Display

Weather values that are unavailable (`null`) now show `—` instead of `0.0°C`.  
Soil fields that are still loading show `🤖 Analyzing...` and update when background enrichment completes.

## 💡 Design Principles

| Principle | Implementation |
|-----------|----------------|
| **Zero Static Defaults** | No static zone tables, no fallback crop lists, no hardcoded market prices. LLM or API — or an honest empty state. |
| **Global Precision** | 195-country ISO list + 100% LLM-driven state/district resolution (24h cache). Any rural location in any country. |
| **Real Climatology** | Open-Meteo Archive API provides real 2-year historical averages per lat/lon — not zone-level approximations. |
| **Search-Grounded AI** | Gemini + Google Search for real-time crop advisories, current market prices, active pest alerts. |
| **Climate Intelligence** | NOAA CPC ONI data (free) reveals El Niño/La Niña; 9 independent climate threat assessors run in parallel. |
| **Graceful Emptiness** | When data is unavailable, the system shows an honest `—` or `🤖 Analyzing...` rather than fabricated fallback values. |
| **Free & Private** | LLaMA 3.2 runs locally via Ollama. NOAA is public. Open-Meteo requires no API key. |

## 📡 100% Dynamic Data Pipeline

### Weather Data (Step 2)

| Priority | Source | What it provides |
|----------|--------|-----------------|
| 1 | **Open-Meteo Current API** | Temperature, humidity, rainfall 7d, wind, UV, soil temp — real-time per lat/lon |
| 2 | **`_llm_estimate_current_weather()`** | Gemini estimates conditions from coordinates + month — no static defaults |
| 3 | **Null state** | Honest `—` shown in UI; crop LLM still runs with climate-only context |

### 6-Month Forecast (Step 3)

| Priority | Source | What it provides |
|----------|--------|-----------------|
| 1 | **Open-Meteo Archive API** | Real 2-year historical monthly averages for this exact lat/lon (temperature, rainfall, humidity) |
| 2 | **`_llm_generate_forecast()`** | Gemini generates forecast from coordinates + country context |
| 3 | **Empty list** | No static zone tables used — ever |

The live temperature is used to compute an **offset** anchoring the archive-based forecast to today's actual conditions for district-level accuracy.

### Soil & Market Enrichment (Step 4)

| Stage | Source | Timing |
|-------|--------|--------|
| **Fast (parallel)** | `_llm_enrich_fast()` — direct Gemini 2.0 Flash Lite call, targets < 8s | Fires simultaneously with weather fetch |
| **Rich (background)** | `/api/enrich-soil` → `_llm_enrich()` — search-grounded Gemini | Fires after dashboard renders; updates soil card smoothly |

If both fail: soil shows `🤖 Analyzing...`, market shows "data being gathered". **No zone-based defaults.**

### Crop Recommendations (Step 5)

| Priority | Method |
|----------|--------|
| 1 | In-memory cache (1h) |
| 2 | Ollama + DuckDuckGo web search |
| 3 | Ollama plain |
| 4 | Gemini + Google Search Grounding |
| 5 | Gemini plain (4-key rotation) |
| 6 | `_llm_simple_fallback()` — minimal Gemini/Ollama call |
| 7 | **Empty list** — no static crop tables |

## 🌐 Climate Signal Intelligence

Version 3.1+ (and v4.0) adds a full **9-Dimensional Climate Intelligence** layer via NOAA + Gemini Search.

### 9 Climate Threat Dimensions

| # | Threat | Source |
|---|--------|--------|
| 1 | **ENSO (El Niño/La Niña)** | NOAA CPC ONI — free, fetched every 6 hours |
| 2 | **Drought Index** | Live 7-day rainfall vs zone norm |
| 3 | **Heat Stress** | Live temperature + zone-specific thresholds |
| 4 | **Frost / Cold Stress** | Live min temperature |
| 5 | **Flood / Excess Rainfall** | Live 7-day rainfall > 2.5× norm |
| 6 | **Cyclone Basin** | 7 global basins, season calendar |
| 7 | **Wildfire Risk** | Temperature + drought + high-risk zones |
| 8 | **Soil Moisture Stress** | Derived from rainfall deficit/surplus |
| 9 | **Climate Change Trend** | Gemini Search Grounding — real-time regional advisories |

### ENSO Adjustment Formula

```text
adjusted_rainfall = baseline_rainfall × (1 + rainfall_factor)
adjusted_temp     = baseline_temp + temp_offset_c
```

Applied to all 6 forecast months via `apply_enso_to_forecast()`.

## 🦙 LLM Integration

### Provider Priority Chain

```
1. Ollama + DuckDuckGo Tool          ← Local model with web search (primary)
2. Ollama Plain                      ← Local model, no internet
3. Gemini + Google Search Grounding  ← Real-time advisories, current prices
4. Gemini Plain (4-key rotation)     ← Fallback if Search Grounding unavailable
5. _llm_simple_fallback()            ← Minimal single-shot prompt (Gemini/Ollama)
6. Empty result                      ← Honest empty state (no static filler)
```

### Gemini Key Rotation

Supports up to **4 Gemini API keys** (`GEMINI_API_KEY` through `GEMINI_API_KEY_4`), rotated automatically on `429 RESOURCE_EXHAUSTED`. Three keys effectively triple the free daily quota.

### Gemini Model Fallback Chain

```
gemini-2.0-flash-lite → gemini-2.5-flash-lite → gemini-2.0-flash
  → gemini-flash-lite-latest → gemini-2.5-flash
```

Fast enrichment uses: `gemini-2.0-flash-lite` (first attempt — fastest)  
Search grounding uses: `gemini-2.0-flash`, `gemini-2.5-flash`, `gemini-2.0-flash-001`

## 🚀 Installation

### Prerequisites
- Python **3.8+**
- [Ollama](https://ollama.com/download) *(recommended — free local LLM)*
- A free **[Google Gemini API key](https://aistudio.google.com/app/apikey)** *(strongly recommended for soil/market/crop data)*

> **Open-Meteo and NOAA require no API key** — fetched automatically.

### 1. Clone the Repository
```bash
git clone https://github.com/tirthch25/AI-Powered-Weather-Resilient-Crop-Advisor.git
cd AI-Powered-Weather-Resilient-Crop-Advisor/agri_crop_recommendation
```

### 2. Create & Activate a Virtual Environment
```bash
python -m venv .venv

# Windows
.venv\Scripts\activate

# macOS / Linux
source .venv/bin/activate
```

### 3. Install Dependencies
```bash
pip install -r requirements.txt
```

### 4. Set Up Ollama (Local LLM — Recommended)
```bash
ollama pull llama3.2        # ~2 GB download, one-time
ollama serve                # Start the local server
```

### 5. Configure Environment Variables
```bash
# Windows PowerShell
Copy-Item .env.example .env

# Linux / macOS
cp .env.example .env
```

Edit `.env`:
```env
LLM_PROVIDER=ollama
OLLAMA_MODEL=llama3.2
OLLAMA_BASE_URL=http://localhost:11434

# Gemini — strongly recommended for soil, market prices, and crop data
GEMINI_API_KEY=your_gemini_api_key_here
GEMINI_API_KEY_2=          # Optional — doubles quota
GEMINI_API_KEY_3=          # Optional — triples quota
GEMINI_API_KEY_4=
```

### 6. Start the Platform
```bash
python run_website.py
```

### 7. Open in Browser
```
http://localhost:8000          ← Web Interface
http://localhost:8000/docs     ← Interactive Swagger API Docs
http://localhost:8000/health   ← System status
http://localhost:8000/climate-signals?country=india  ← ENSO + 9-threat status
```
